One question comes up a lot, but I've yet to find it actually clearly documented and explained:

What exactly is the different between patching Solaris release up to a certain release level and actually upgrading Solaris to that level?

- patching only updates what's installed on the system, it won't install new pkgs or remove old obsolete and unused pkgs. There have been a few exceptions to this rule, but it's very rare.
- patching doesn't update SUNWsolnm so /etc/release (and other places) are not updated. Many applications check these locations to check your Solaris release to ensure you meet their minimum requirements
- new hardware often requires pkgs which are not provided by patching

What's new documents lists new things and enhancements, but doesn't differentiate between those introduced by patches and those introduced by the upgrade alone.

Examples:

- Solaris 10u7 clean install SUNWCXall vs 10u9 clean install SUNWCXall
- fully patched s10u7 compare to fully patched s10u9
Solaris 10u7 + u9 Patch Update Set compare to clean s10u9
