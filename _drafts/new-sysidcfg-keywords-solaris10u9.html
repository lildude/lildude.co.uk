With the release of Solaris 10 09/10 (aka S10u9) today, a lot of people will be eager to update, but before you update your jumpstart images and get updating, I thought I'd give you all a heads-up on some new keywords for sysidcfg which you may only discover when your jumpstarts suddenly start going interactive.

With integration of changes for CR 6952572 and CR 6952415 into S10U9_b10, the following keywords have been introduced to sysidcfg(4) to conform to the corresponding changes with respect to MyOracle Auto Registration for Solaris (PSARC/2010/134)

If the needed parameters are not specified, the Custom Jumpstart may fall back to the interactive install to get the input from the users.

Please see the last section for changes to be done to sysidcfg(4) to make the Custom Jumpstart not to go interactive.

1. New sysidcfg(4) Keywords:

The following keywords have been introduced to sysidcfg(4)
* auto_reg with values
* anon
* noproxy
* all
* none
* disable
* oracle_user
* oracle_pw
* http_proxy_host
* http_proxy_port
* http_proxy_user
* http_proxy_pw

2. Details:
auto_reg: Used to specify the parameters needed for MyOracle Automatic Registration.
anon
o anonymous auto registration; MyOracle user/passwd ignored.
o if MyOracle keywords specified, they will be ignored.
o proxy keywords expected. If not specified in sysidcfg, resort to screens.

noproxy
o no proxy server information is expected
o if proxy keywords not specified, they will be ignored
o MyOracle keywords expected. If not specified, resort to screens.

all o both MyOracle and proxy keywords are expected, either through sysidcfg or by screens

none o anonymous registration with no proxy information

disable o The auto-registration parameters are not expected. Any data provided/ not provided will be ignored.

oracle_user:
MyOracle username as string

oracle_pw:
MyOracle password (clear text)

http_proxy_host:
a valid hostname of Proxy Serveras string

http_proxy_port:
a valid port of Proxy Server as number

http_proxy_user:
proxy username of Proxy Server as string

http_proxy_pw:
password (clear text) for Proxy Server

typical way of sysidcfg entry for auto configuration:
auto_reg=[anon|noproxy|all|none|disable] {
oracle_user=username1
oracle_paswd=password123
http_proxy_user=username
http_proxy_pw=password123
http_proxy_host=sss.com
http_proxy_port=8080
}
If any of the above 6 parameters is needed to have no value optionally, then a null-string "", should be specified in order
for install not to go interactive.

3. Examples:
a. Anonymous auto registration
auto_reg=anon {
http_proxy_user=username
http_proxy_pw=password123
http_proxy_host=host.com
http_proxy_port=8080
}

b. Not anonymous, all of proxy information should be specified
auto_reg=yes {
oracle_user=username1
oracle_pw=skd1sk$swsk2
http_proxy_host=host.com
http_proxy_port=8080
http_proxy_user=username2
http_proxy_pw=password123
}

c. No proxy information needed to be specified
auto_reg=noproxy {
oracle_user=username1
oracle_pw=passwd1
}

d. Not anonymous, but only proxy host and port needs to be specified. Proxy doesn't need user/password
Note: The screen should be avoided as unspecified parameters are intended to be optional and the
process is intended to be non-interactive.
auto_reg=all {
oracle_user=username1
oracle_pw=passwd1
http_proxy_host=host.com
http_proxy_port=8080
http_proxy_user=""
http_proxy_pw=""
}

e. Anonymous registration with no proxy
auto_reg=none

f. No auto-registration. Do not expect any parameter.
auto_reg=disable

4. Information for the internal development environment
If any of the auto_reg modes like anon, yes, noproxy and all the required parameters are not specified, the install will fallback to the interactive install.
To avoid this,
Either add the following line in sysidcfg(4):
auto_reg=none or auto_reg=disable
or
In any selected mode, specify the null-string as the value for the required parameter.

More details can be found at http://docsview.sfbay.sun.com/app/docs/doc/817-1985/autoreg-1?a=view
